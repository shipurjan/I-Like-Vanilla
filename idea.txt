What you’re seeing in **I Like Vanilla** is actually a **very typical Minecraft shadow-map pipeline**, just with a few stylistic filters layered on top. Those settings tell us quite a lot about how it works internally.

Let’s unpack the interesting parts.

---

# Why Minecraft shader shadows look “square” sometimes

In **Minecraft** shaders, shadows are usually rendered into a **shadow map texture**.

Simplified:

```
world → rendered from sun → depth texture (shadow map)
```


Then during lighting:


```
compare pixel depth vs shadow map depth

```

If the shadow map resolution is low relative to the world scale, you get **blocky edges**.

Example mental model:


```
shadow map resolution = 1024×1024
world area covered = 200 blocks
```


So one shadow pixel covers about:

```
~0.2 blocks
```

If the coverage increases:

```
world area = 800 blocks
```

Then one shadow pixel covers:

```
~0.8 blocks
```

Now shadows look **chunky**.

That’s why shaders expose settings like:

* **Shadow map scale**
* **Shadow map resolution**
* **Shadow map distance**

They control **how many world units one shadow texel represents**.

---

# What “Pixelated Shadows” is doing

The **Pixelated Shadows** setting in **I Like Vanilla** likely does something like:

```
shadow_uv = floor(shadow_uv * grid) / grid
```


That snaps the sampling coordinates to a grid.

Result:

```
shadow map
→ quantized sampling
→ blocky shadow edges
```

Which matches the **Minecraft aesthetic**.

---

# Why “Shadow Filtering” doesn’t work with Pixelated Shadows

The option description you pasted explains something important:

> “Shadow Filtering works best with Temporal Filter enabled, has no effect when pixelated shadows is enabled”

That means:


```
Shadow Filtering = blur / PCF
Pixelated Shadows = quantized sampling
```

These conflict.

If you blur the shadow first, then quantize it, the blur disappears.


So the shader disables filtering when pixelation is active.

# Your idea: physically correct penumbra + pixel shadows

Now to your **question about point 5**.

You asked:

> Would I need PCSS for that?

Yes — **PCSS is the best candidate**.


Because PCSS already computes **penumbra width based on blocker distance**.

Conceptually:

```
penumbra size ∝ distance(light, blocker)
```


Meaning:


```
near contact → sharp

far shadow → soft
```

Exactly the effect you want.

# Why the current shader can't produce penumbra

From the settings you showed:

* **Shadow Filtering**
* **Noise Amount**
* **Temporal Filter**


This strongly suggests it uses:

```
PCF (percentage closer filtering)
+ temporal accumulation
```

PCF only does:

```

blur shadow edges
```

It **does not compute penumbra size physically**.

So shadows stay roughly the same softness everywhere.
